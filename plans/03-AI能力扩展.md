# AI 能力扩展建议

## 现有 AI 能力梳理

| 能力 | 实现 | 现状 |
|------|------|------|
| 图片 OCR | SiliconFlow Vision (GLM-4.6V) | 准确率约 85-90% |
| 文字报告 OCR | 通义千问 (DashScope) | 结构化提取 |
| 检验项名称标准化 | 算法引擎 + LLM 混合 | 准确率约 90% |
| 报告分组分类 | LLM | 正常运行 |
| AI 智能解读 | Gemini (pucode.com) | 单报告/多报告/趋势 |
| 健康评估 | LLM | 综合分析 |

---

## A. 新 AI 功能建议

### 1. 患者预测性健康风险评分

**功能描述**：基于历史检验数据、用药记录、体温趋势，用时序分析预测指标恶化趋势，计算 0-100 综合风险分数。

**技术实现思路**：
- 输入特征：过去 N 次检验值、异常指标数量、危急值频次、指标变化速率
- 模型方案：
  - 轻量级：规则打分（各异常指标加权求和）→ 可立即上线
  - 增强版：时序回归模型（Python 微服务，Rust 调用）
- 风险分级：低危（0-30）/ 中危（31-70）/ 高危（71-100）
- 实时预警：风险分数突破阈值时触发多通道告警

**预期价值**：提前 3-6 周识别健康恶化趋势，降低危急事件发生率

---

### 2. 多模态医学影像 AI 分析

**功能描述**：支持 X 光、CT 影像的自动化分析与异常定位，结合检验报告数据进行影像-检验关联分析。

**技术实现思路**：
- 集成医学影像 API（阿里云医学影像、百度医学图像识别）
- 生成结构化影像报告摘要
- 与现有 OCR 流程无缝衔接：影像报告 → 结构化数据 → 趋势分析

**预期价值**：缩短影像报告解读时间 50%+，支持基层医疗机构远程诊疗

---

### 3. 用药-检验关联推理引擎

**功能描述**：建立用药与检验指标变化的因果映射，分析特定用药对各指标的预期影响，识别用药不良反应。

**技术实现思路**：
- 构建"用药 → 指标变化"知识图谱（结合 LLM + 药理数据库）
- 时间轴对齐：用药开始时间 vs 指标变化时间点
- 生成用药效果评估报告（预期效果 vs 实际效果）

**预期价值**：帮助医生评估用药方案有效性，提高用药安全

---

### 4. RAG 医学知识问答助手

**功能描述**：基于患者自身数据构建私域知识库（RAG），支持自然语言提问，LLM 基于患者历史数据生成精准回答。

**技术实现思路**：
- 构建患者级别的报告摘要向量索引（本地向量库如 Qdrant）
- 用户提问 → 检索相关报告/指标 → 注入 LLM context → 生成回答
- 隐私保护：不传患者身份信息，仅传检验指标值和时间序列

**示例问答**：
- "为什么我的肌酐一直偏高？" → 分析近 6 次肌酐趋势 + 用药关联
- "血糖改善了吗？" → 对比最近 3 次与半年前的血糖值

**预期价值**：70% 常见疑问自动解答，减轻医生咨询压力

---

### 5. 异常指标聚类与疾病模式识别

**功能描述**：识别相似患者的异常指标聚类，当患者出现某个已知"异常指标组合模式"时，自动提示可能的疾病方向。

**技术实现思路**：
- 用无监督学习（K-means / DBSCAN）对患者异常指标组合进行聚类
- 建立"异常模式库"：白细胞↑ + 中性粒细胞↑ + CRP↑ → 疑似细菌感染
- 与 ICD-10 疾病编码关联，提供诊疗路径建议

**预期价值**：支持早期疾病筛查，辅助基层医生快速定位诊疗方向

---

### 6. 费用-疗效关联分析

**功能描述**：分析医疗费用投入与检验指标改善的关系，识别"高效治疗项目"与"低效费用"。

**技术实现思路**：
- 建立费用时间轴与指标变化时间轴的对齐模型
- LLM 分析特定治疗阶段的费用构成与指标改善关联
- 生成费用-效能报告（投入产出比）

**预期价值**：帮助患者优化治疗决策，机构降低无效费用

---

### 7. 自适应个性化健康建议

**功能描述**：基于患者年龄、性别、体质、既往史、生活方式，生成个性化的运动/饮食/作息建议，随指标变化动态调整。

**技术实现思路**：
- 多维度患者画像构建（生命体征 + 检验数据 + 生活方式问卷）
- LLM 生成个性化建议，相同指标异常对不同体质患者给出不同建议
- 建议有效性追踪：下次复查指标改善则强化该类建议

**预期价值**：个性化建议执行率比通用建议高 40%+

---

### 8. 中医体质辨识融合

**功能描述**：将西医检验数据映射到中医体质诊断框架，提供中西医结合的调理建议。

**技术实现思路**：
- 中医九种体质分类（平和质、气虚质、阴虚质等）与检验指标映射规则
- 结合体质类型生成食疗、穴位、草药等调理建议
- 与西医用药建议并列展示（"西医治疗建议 | 中医调理建议"）

**预期价值**：契合国内患者习惯，差异化竞争优势

## B. 现有 AI 功能优化

### 1. OCR 准确率提升

**现状问题**：
- 手写内容识别率低
- 表格格式报告易错位
- 低质量/模糊照片识别失败

**优化方向**：
- 图像预处理：上传时自动矫正倾斜、增强对比度、去噪
- 低置信度字段自动标记（橙色高亮），提示医生复核
- 建立 OCR 错误反馈机制：医生标注错误 → 积累样本 → 优化 Prompt
- 多模型竞赛：同一图片同时调用 2 个 OCR 模型，取置信度更高的结果

**目标**：准确率从 85-90% → 95%+

---

### 2. 检验项名称标准化精准度提升

**现状问题**：
- 非规范缩写（"HB"识别为血红蛋白）有时漏掉
- 不同医院对同一指标的叫法不同

**优化方向**：
- 构建"医院级别自定义字典"：记录每家医院的习惯命名
- 患者级别学习：同一患者连续报告中确认过的名称自动记忆
- 医生标注反馈：标注标准化错误 → 实时更新本地字典

**目标**：准确率从 90% → 97%+

---

### 3. 智能解读医学准确性增强

**现状问题**：LLM 可能产生医学错误或过度诊断。

**优化方向**：
- 集成医学知识图谱验证：关键医学陈述与知识库交叉核对
- 强化免责声明分级：AI 置信度低时显示更明显的"请咨询医生"提示
- 医生审核反馈库：医生修改 AI 解读后的内容用于优化 Prompt
- 与最新医学指南对齐（定期更新 System Prompt 中的指南参考）

---

### 4. 趋势分析统计学增强

**现状问题**：当前趋势解读基于 LLM 定性分析，缺乏统计学严谨性。

**优化方向**：
- 增加统计显著性检验（p 值计算）
- 变化率标注：同比增长率、环比下降率
- 异常值检测：识别可能的采样错误（单次极端值）
- 趋势线拟合：线性/多项式拟合、LOESS 平滑
- 预测置信区间：下次检验结果的预期范围（灰色区间）

---

### 5. 危急值告警可靠性

**现状问题**：Dashboard 危急值告警已移除（显示异常），告警链路不完整。

**优化方向**：
- 重建危急值告警：从报告保存时同步触发，写入 `alerts` 表
- 多通道推送：应用内弹窗 → 邮件 → 短信（逐级升级）
- 告警响应时间记录：追踪医疗质量指标
- 假阳性过滤：学习历史误报特征，减少干扰

---

### 6. 健康评估完整性

**现状问题**：健康评估维度较单一，仅基于检验数据。

**优化方向**：
- 纳入生活方式问卷（吸烟/饮酒/运动/睡眠）
- 支持家族史影响因素
- 动态更新：每次新报告后自动重新评估
- 对标分析：与同年龄段同性别人群健康指标对比（百分位数）
